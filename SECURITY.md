# Security Policy

## Supported Versions

Only the latest GitHub Release is actively supported for security fixes.

## Reporting a Vulnerability

Please do not open a public issue for vulnerabilities or sensitive reports.

This project does not currently publish a dedicated security email address. To report a security issue, contact the maintainer privately through GitHub first and include:

- Affected version or commit.
- Steps to reproduce.
- Expected and actual behavior.
- Any proof-of-concept files or screenshots needed to understand the impact.

The maintainer will review the report and coordinate a fix before public disclosure when appropriate.
